# 📚 Lexicon (Лексикон)

**Lexicon** — это современное и интуитивно понятное Android-приложение для ведения персонального словаря. Проект разработан как демонстрация современных практик разработки на стеке Jetpack Compose и Clean Architecture.

![Kotlin](https://img.shields.io/badge/kotlin-2.4.0-blue.svg)
![Compose](https://img.shields.io/badge/compose-material3-green.svg)

## ✨ Основные возможности

*   **Управление определениями:** Полный цикл CRUD (создание, чтение, редактирование, удаление) ваших записей.
*   **Визуальный контекст:** Возможность прикреплять изображения к словам. Фотографии автоматически сохраняются во внутреннее хранилище приложения для обеспечения постоянного доступа.
*   **Мгновенный поиск:** Быстрая фильтрация слов с поддержкой алфавитных заголовков (A-Z/А-Я) для удобной навигации по словарю.
*   **Избранное:** Возможность отмечать важные слова для быстрого доступа в отдельной вкладке.
*   **Персонализация:** Поддержка светлой и темной тем оформления, которая сохраняется между сессиями.
*   **Производительность:** Полностью реактивный интерфейс на основе потоков данных (Flow), обеспечивающий мгновенный отклик на действия пользователя.

## 🛠 Технологический стек

Проект построен на передовых инструментах разработки Android (состояние на июль 2026 года):

*   **Язык:** [Kotlin 2.4.0+](https://kotlinlang.org/)
*   **UI:** Jetpack Compose (Material 3)
*   **Архитектура:** Clean Architecture (Domain, Data, Presentation) + MVVM.
*   **DI:** Hilt (Dagger) для внедрения зависимостей.
*   **База данных:** Room (с поддержкой KSP и Flow) для надежного локального хранения.
*   **Навигация:** Type-Safe Navigation Compose (безопасная типизация маршрутов).
*   **Хранение настроек:** Jetpack DataStore (Preferences).
*   **Работа с изображениями:** Coil для асинхронной загрузки и кэширования.
*   **Асинхронность:** Kotlin Coroutines & Flow (StateFlow, SharedFlow).

## 🏗 Архитектура

Приложение следует принципу **Unidirectional Data Flow (UDF)**:
1.  **Domain:** Чистая бизнес-логика (Use Cases и интерфейсы репозиториев).
2.  **Data:** Реализация репозиториев, работа с Room DAO и файловой системой.
3.  **Presentation:** ViewModels обрабатывают команды (Commands) и преобразуют потоки данных в состояния экрана (States) и разовые события (Events).

## 🚀 Планы по развитию

- [ ] **Категории и теги:** Группировка слов по темам (например, "Работа", "Путешествия").
- [ ] **Резервное копирование:** Экспорт и импорт базы данных в формате JSON или CSV.
- [ ] **Интеграция с API:** Автоматическое получение определений и синонимов из внешних онлайн-словарей.
- [ ] **Режим обучения:** Карточки (Flashcards) для эффективного запоминания добавленных слов.
- [ ] **Произношение:** Добавление аудио-дорожек к словам с помощью Text-to-Speech.
